## 1. 项目概述

### 1.1 简介

**NutBot** 是一个 AI 驱动的跨平台自动化助手，类似于钢铁侠的 Jarvis。它能够：

- 🌐 **浏览器自动化** - 自动打开网页、填写表单、点击按钮、提取数据
- 🖥️ **桌面控制** - 操控本地应用（微信、QQ、记事本等），模拟键盘鼠标
- 📸 **屏幕理解** - 截图分析，理解界面内容，精确定位元素
- ⚡ **命令执行** - 运行系统命令，管理文件
- ⏰ **定时任务** - 设置定时任务，自动执行
- 🧠 **记忆系统** - 记住用户偏好和习惯
- 💻 **Web UI** - 现代化界面，支持移动端

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 多 Agent 协作 | Analyzer（分析）、Executor（执行）、Reviewer（检验）三 Agent 协作 |
| 多 Provider 支持 | 支持 OpenAI、Anthropic 及兼容接口 |
| MCP 协议 | 支持 Model Context Protocol，可扩展外部工具 |
| Skills 系统 | 通过 Markdown/JSON 文件扩展 Agent 能力 |
| 视觉能力 | 支持截图分析、OCR、元素定位（SoM） |

### 1.3 技术栈

- **后端**: Node.js + TypeScript + Fastify
- **前端**: Vue 3 + Tailwind CSS + Vite
- **浏览器自动化**: Playwright
- **桌面自动化**: nut.js + 平台原生 API
- **AI 接口**: OpenAI SDK、Anthropic SDK

## 4. 核心模块详解

### 4.1 Gateway（网关层）

**文件**: [`src/gateway/index.ts`](src/gateway/index.ts:1)

Gateway 是 NutBot 的中央控制器，负责协调所有子系统：

```typescript
class Gateway {
  config: ConfigManager          // 配置管理
  providerManager: ProviderManager  // AI Provider 管理
  toolRegistry: ToolRegistry     // 工具注册中心
  agent: Agent                   // AI Agent
  sessionManager: SessionManager // 会话管理
  agentProfiles: AgentProfileManager // Agent 配置
  cronManager: CronManager       // 定时任务
  server: Server                 // HTTP 服务器
}
```

**初始化流程**:
1. 加载配置
2. 初始化日志系统
3. 初始化 Provider 管理器
4. 初始化工具注册中心
5. 初始化 Agent
6. 初始化会话管理
7. 初始化定时任务
8. 启动 HTTP 服务器

### 4.2 Agent（AI 核心）

**文件**: [`src/agent/index.ts`](src/agent/index.ts:1)

Agent 是 NutBot 的 AI 决策核心，负责：
- 接收用户输入
- 调用 AI 模型进行推理
- 解析工具调用
- 执行工具并获取结果
- 生成最终回复

**核心方法**:
- `chat(message, options)` - 主对话方法
- `interrupt(reason)` - 中断当前任务
- `detectModelCapabilities()` - 检测模型能力

**执行循环**:
```
1. 接收用户消息
2. 构建上下文（历史消息 + 系统提示 + 工具描述）
3. 调用 AI 模型
4. 解析响应（工具调用或最终结果）
5. 如需要工具调用 → 执行工具 → 返回结果 → 继续步骤 3
6. 生成最终回复
```

### 4.3 Session（会话管理）

**文件**: [`src/agent/session.ts`](src/agent/session.ts:1)

会话管理负责：
- 创建、读取、更新、删除会话
- 消息历史存储
- 会话元数据管理

**会话结构**:
```typescript
interface Session {
  id: string
  title: string
  createdAt: string
  updatedAt: string
  messages: ChatMessage[]
  metadata?: Record<string, unknown>
}
```

### 4.4 Provider（AI Provider）

**文件**: 
- [`src/providers/index.ts`](src/providers/index.ts:1) - Provider 管理器
- [`src/providers/openai.ts`](src/providers/openai.ts:1) - OpenAI Provider
- [`src/providers/anthropic.ts`](src/providers/anthropic.ts:1) - Anthropic Provider

支持多 Provider 配置，可动态切换模型。

**Provider 配置**:
```typescript
interface ProviderConfig {
  id: string
  name: string
  type: 'openai' | 'anthropic'
  baseUrl?: string
  apiKey?: string
  models?: string[]
  supportsVision?: boolean
}
```

### 4.5 ToolRegistry（工具注册中心）

**文件**: 
- [`src/tools/index.ts`](src/tools/index.ts:1)
- [`src/tools/registry.ts`](src/tools/registry.ts:1)

工具注册中心负责：
- 注册所有可用工具
- 生成工具描述（用于系统提示词）
- 执行工具调用

**工具基类**:
```typescript
abstract class BaseTool {
  abstract name: string
  abstract description: string
  abstract parameters: object
  abstract execute(args: object): Promise<ToolExecuteResult>
}
```

### 8.3 添加新工具

1. 在 `src/tools/` 创建新文件，继承 `BaseTool`:

```typescript
import { BaseTool, type ToolExecuteResult } from './registry.js';

export class MyTool extends BaseTool {
  name = 'mytool';
  description = '我的工具描述';
  parameters = {
    type: 'object',
    properties: {
      action: {
        type: 'string',
        enum: ['doSomething'],
        description: '操作类型'
      },
      param1: {
        type: 'string',
        description: '参数1'
      }
    },
    required: ['action']
  };

  async execute(args: { action: string; param1?: string }): Promise<ToolExecuteResult> {
    try {
      // 执行逻辑
      return { success: true, data: result };
    } catch (error) {
      return { success: false, error: error.message };
    }
  }
}
```

2. 在 `src/tools/index.ts` 中导出:

```typescript
export { MyTool } from './mytool.js';
```

3. 在 `src/tools/registry.ts` 中注册（如果使用动态注册则自动加载）

### 8.4 添加新 Provider

1. 在 `src/providers/` 创建新文件，继承 `BaseProvider`:

```typescript
import { BaseProvider } from './base.js';

export class MyProvider extends BaseProvider {
  type = 'myprovider';
  
  async chat(options) {
    // 实现聊天逻辑
  }
}
```

2. 在 `src/providers/index.ts` 中注册

### 8.5 添加 Skills

在 `skills/` 目录下创建 `.md` 或 `.json` 文件：

**Markdown 格式** (`skills/my-skill.md`):
```markdown
# 我的技能

这是技能描述...

## 使用场景

- 场景1
- 场景2
```

**JSON 格式** (`skills/my-skill.json`):
```json
{
  "name": "my-skill",
  "description": "技能描述",
  "prompt": "详细的 prompt 内容..."
}
```

## 9. 部署说明

### 9.1 本地部署

```bash
# 克隆项目
git clone https://github.com/N0tsLabs/NutBot.git
cd NutBot

# 安装依赖
yarn install
cd web && yarn install && cd ..

# 配置环境变量
# 编辑 ~/.nutbot/config.json 或在项目根目录创建 .env

# 启动开发服务器
yarn dev
```

访问 http://localhost:18800

### 9.2 生产部署

```bash
# 构建
yarn build

# 启动生产服务器
yarn start
```

### 9.3 Docker 部署（可选）

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build
EXPOSE 18800
CMD ["npm", "start"]
```

### 9.4 配置说明

首次运行需要配置 AI Provider：

1. 打开 Web UI http://localhost:18800
2. 进入设置页面
3. 添加 Provider（填入 API 地址和 Key）
4. 选择默认模型

或在配置文件中直接配置：

```json
{
  "providers": {
    "openai": {
      "id": "openai",
      "name": "OpenAI",
      "baseUrl": "https://api.openai.com/v1",
      "apiKey": "your-api-key",
      "type": "openai",
      "models": ["gpt-4", "gpt-3.5-turbo"],
      "defaultModel": "gpt-4"
    }
  }
}
```
