# NutBot 需求文档

> 版本：v2.0
> 日期：2026-02-27
> 状态：重构中

---

## 1. 核心目标

打造一个**让用户只需动嘴，AI 帮你搞定一切**的智能助手。

**核心原则**：
- AI 自己判断需要什么数据（不写死在代码里）
- 任务必须完全完成才能回复用户
- 每次工具执行后，AI 问自己：用户要的东西得到了吗？

---

## 2. 多 Agent 协作架构

### 2.1 三大 Agent

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **Analyzer** | 分析任务，拆解步骤 | 用户指令 | 任务步骤列表 + 成功标准 |
| **Executor** | 执行工具，收集结果 | 步骤列表 | 执行结果 |
| **Reviewer** | 检验结果，判断完成 | 执行结果 vs 成功标准 | 通过/重试/完成 |

### 2.2 工作流程

```
用户: "打开B站搜索影视飓风"

Analyzer → 拆解步骤:
  1. goto B站
  2. snapshot (获取页面)
  3. click 搜索框
  4. type 关键词
  5. press Enter
  6. snapshot (获取搜索结果)
  成功标准: 返回视频列表

Executor → 依次执行:
  - goto → AI判断需要snapshot → 执行
  - click → type → press Enter
  - press后 → AI判断页面变化了 → snapshot
  - 分析内容 → 提取视频列表

Reviewer → 检验:
  - 有视频列表吗? → 通过
  - 生成最终回复

用户收到: "已找到以下视频：1. xxx 2. xxx ..."
```

### 2.3 AI 决策规则（核心）

```
每次工具执行后，问自己：
1. 页面是否发生变化？（URL变化、新内容加载）
2. 用户要的东西得到了吗？
3. 还需要继续获取什么数据？

需要新数据 → 继续执行工具
数据已完整 → 生成回复，任务完成
```

## 3. 桌面操作实现思路

### 3.1 核心原理

**与浏览器操作相同**：把界面转换为结构化元素树 → AI 解析 → 执行操作

### 3.2 技术栈

| 平台 | 技术 | 用途 |
|------|------|------|
| Windows | nut.js + UI Automation | 桌面自动化 + 元素定位 |
| macOS | nut.js + Accessibility API | 桌面自动化 + 元素定位 |
| Linux | nut.js + AT-SPI2 | 桌面自动化 + 元素定位 |

### 3.3 元素树结构

```typescript
interface DesktopElement {
  id: string;           // 唯一标识
  type: string;         // 元素类型：window, button, input, text, etc.
  name?: string;        // 元素名称/文本
  role?: string;        // 角色：menu, toolbar, etc.
  bounds?: { x, y, w, h }; // 边界矩形
  children?: DesktopElement[]; // 子元素
  clickable: boolean;   // 是否可点击
  editable: boolean;    // 是否可编辑
}
```

### 3.4 操作流程

```
1. screenshot → 获取屏幕截图
2. desktop_snapshot → 获取桌面元素树
3. AI 分析元素树 → 确定目标元素
4. click [element_id] → 点击元素
5. type [element_id] "文字" → 输入文字
6. screenshot → 验证操作结果
```

### 3.5 与浏览器操作的异同

| 维度 | 浏览器 | 桌面 |
|------|--------|------|
| 元素获取 | Playwright DOM | nut.js + 平台API |
| 元素树 | DOM 树 | Accessibility 树 |
| 相同点 | 都是元素树 → AI解析 → 执行操作 |  |

## 4. 用户交互模式

| 模式 | 说明 | 示例 |
|------|------|------|
| **直接对话** | 用户发指令 | "帮我搜索B站影视飓风" |
| **状态展示** | 系统展示进度 | 工具调用详情、当前步骤 |

**状态展示内容**：
- 当前任务步骤（进度 2/6）
- 工具调用（参数 + 结果）
- 检验结果（通过/重试中）

---

## 5. 上下文管理

- **动态获取**：AI 自主判断需要什么上下文
- **智能压缩**：超过阈值时压缩已完成步骤
- **保留关键**：每个步骤的结果摘要保留

## 8. 与旧版本的区别

| 旧版本 | v2.0（当前设计） |
|--------|-----------------|
| snapshot 智能过滤 | snapshot 不过滤，直接给全部内容 |
| 限制元素数量 | 不限制，让 AI 自己判断 |
| 代码写死某些逻辑 | AI 自主决策 |
| 单一 Agent | 三 Agent 协作 |

## 9. 验收标准

**功能**：
- [ ] AI 能自己判断需要获取什么数据
- [ ] 搜索后能返回结果列表，而不是说"请查看"
- [ ] 任务完全完成才回复用户

**体验**：
- [ ] 简洁的状态展示
- [ ] 用户只看到结果，不看到迭代日志
